## 🎯 一、产品定位

### 1.1 产品愿景
**"每一本书都有它的读者，每一个问题都有它的答案。"**

天一阁（SkyOne·书阁）是一个**个人知识管理 + 智能问答系统**，帮助用户：
- 构建私有数字图书馆
- 实现知识的语义化检索
- 通过 RAG 技术获得基于个人藏书的智能问答

### 1.2 目标用户
- **重度阅读者**：拥有大量电子书，需要高效管理
- **研究者/学生**：需要快速检索文献和笔记
- **知识工作者**：希望建立个人第二大脑

### 1.3 核心价值
| 价值 | 说明 |
|------|------|
| 📚 统一管理 | 多格式电子书统一入库 |
| 🔍 语义检索 | 基于向量的相似度搜索，超越关键词匹配 |
| 💬 智能问答 | RAG 技术，答案来源于你的藏书 |
| 🔌 可扩展 | MCP 协议，可接入外部数据源和工具 |

## 🏗️ 三、信息架构

### 3.1 核心实体

```
Document（文档）
├── id: UUID
├── title: str
├── authors: List[str]
├── description: str
├── publisher: str?
├── published_date: date?
├── subjects: List[str]
├── language: str = "zh"
├── format: DocumentFormat (PDF/TXT/MD)
├── file_path: str
├── file_size: int
├── page_count: int?
├── created_at: datetime
├── updated_at: datetime
├── processing_status: ProcessingStatus
└── processing_error: str?

Chunk（文本块）
├── id: UUID
├── document_id: UUID
├── content: str
├── chunk_index: int
├── page_number: int?
├── tokens: int?
├── embedding: Vector?
└── metadata: Dict

Library（图书馆）
├── id: UUID
├── name: str
├── description: str?
├── documents: List[Document]
├── created_at: datetime
└── updated_at: datetime
```

### 3.2 处理流程

```
用户上传文档
    ↓
文档解析（PDF/TXT/MD）
    ↓
文本提取 + 元数据提取
    ↓
文本分块（Chunking）
    ↓
向量化（Embedding）
    ↓
存储到向量数据库
    ↓
用户提问
    ↓
问题向量化
    ↓
相似度搜索（Top-K）
    ↓
构建 Prompt（Context + Question）
    ↓
LLM 生成回答
    ↓
返回答案 + 来源引用
```

## 🎨 四、用户界面 (UI MVP)

### 4.1 页面结构

```
/ (首页)
├── 导航栏（Logo + 搜索 + 上传按钮）
├── 统计卡片（文档数、Chunk 数、今日问答）
├── 最近入库文档
└── 快捷入口

/library（图书馆）
├── 文档列表（网格/列表视图）
├── 筛选（格式、语言、日期）
├── 排序（名称、日期、大小）
└── 批量操作

/documents/{id}（文档详情）
├── 元数据展示
├── 内容预览
├── 相关文档推荐
└── 删除/重新处理按钮

/search（搜索）
├── 搜索框
├── 搜索结果列表
└── 过滤器

/chat（智能问答）
├── 对话历史
├── 输入框
├── 知识库选择
└── 答案展示 + 来源引用

/settings（设置）
├── 模型配置
├── 向量数据库配置
├── 分块策略
└── API 密钥管理
```

### 4.2 设计风格
- **风格**: 现代简约，卡片式布局
- **主色调**: 深蓝色 (#1e3a8a) + 金色 (#f59e0b)
- **字体**: Inter 无衬线
- **交互**: 拖拽上传、实时搜索、流式输出
