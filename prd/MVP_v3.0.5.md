## 🎯 一、产品定位与核心功能迭代

### 1.1 RAG 问答体验
当用户在“天一阁”中提问时：
1. **意图理解**：系统自动分析问题，提取关键词并识别问题类型（概念解释、操作指南等）。
2. **多路召回**：基于混合搜索策略，从向量数据库（Qdrant）和关系型数据库中召回相关文档片段。
3. **精准重排序**：通过简单匹配或 Cross-Encoder 模型对召回结果进行打分排序。
4. **生成式回答**：LLM 基于筛选后的 Top-K 文档生成结构化回答，并附带置信度和推导过程。

### 1.2 高级搜索 (Hybrid Search)
- **向量搜索 (Vector Search)**：基于语义相似度匹配文档内容。
- **关键词搜索 (Keyword Search)**：基于标题和描述等元数据进行精确匹配。
- **条件过滤 (Metadata Filtering)**：支持按分类、时间、文件类型等多维条件进行数据过滤。

## 🏗️ 二、数据结构 (Schemas)

核心数据模型包括：

### 2.1 搜索查询 (SearchQuery)
- **query**: 搜索关键词
- **search_type**: vector / keyword / hybrid
- **filters**: 元数据过滤器 (分类、标签、日期等)
- **enable_rerank**: 是否开启二次重排

### 2.2 RAG 响应 (RAGResponse)
- **answer**: 最终生成的答案
- **sources**: 引用的文档来源列表 (包含 ID、标题、内容片段、相关性分数)
- **confidence**: 答案置信度 (0.0 - 1.0)
- **reasoning**: LLM 推理过程的结构化输出
