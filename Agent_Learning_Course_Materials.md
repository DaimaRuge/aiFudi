# Agent Learning 课程材料

基于 learn.shareai.run 网站的 Claude Code 渐进式学习课程

## 课程大纲

### L1: Tools & Execution (工具与执行)
**目标**: 掌握 Agent 基础能力，理解工具执行机制
- s01: The Agent Loop - Bash is All You Need (84 LOC)
  - 最小 Agent 内核：while 循环 + 一个工具
- s02: Tools - One Handler Per Tool (120 LOC)
  - 循环保持不变，新工具注册到调度映射中

### L2: Planning & Coordination (规划与协调)
**目标**: 学习如何组织工作，从待办列表到依赖感知任务板
- s03: Todo - Write Plan Before You Act (176 LOC)
  - 无计划的 Agent 会漂移：先列出步骤，再执行
- s04: Subagents - Clean Context Per Subtask (151 LOC)
  - 子 Agent 使用独立的消息上下文，保持主对话清洁
- s05: Skills - Load on Demand (187 LOC)
  - 按需通过 tool_result 注入知识，不在系统提示中预先加载
- s07: Tasks - Task Graph + Dependencies (207 LOC)
  - 基于文件的任务图，包含排序、并行性和依赖关系

### L3: Memory Management (内存管理)
**目标**: 在上下文限制内保持记忆，实现无限会话
- s06: Compact - Three-Layer Compression (205 LOC)
  - 三层压缩策略使无限会话成为可能

### L4: Concurrency (并发)
**目标**: 掌握非阻塞执行，实现并行工作
- s08: Background Tasks - Background Threads + Notifications (198 LOC)
  - 在后台运行慢操作，Agent 继续向前思考

### L5: Collaboration (协作)
**目标**: 实现多 Agent 协作，建立自主团队
- s09: Agent Teams - Teammates + Mailboxes (348 LOC)
  - 通过异步邮箱委托给持久队友
- s10: Team Protocols - Shared Communication Rules (419 LOC)
  - 一个请求-响应模式驱动所有团队协商
- s11: Autonomous Agents - Scan Board, Claim Tasks (499 LOC)
  - 队友扫描任务板并自行认领任务
- s12: Worktree + Task Isolation - Isolate by Directory (694 LOC)
  - 每个 Agent 在自己的目录中工作

## 课程讲义

### 讲义 1: Agent 基础架构
**核心概念**:
- Agent Loop: 持续监听-响应循环
- 工具调度: 统一接口调用外部能力
- 上下文管理: 保持对话连贯性

**代码模式**:
```python
while True:
    # 1. 接收用户输入
    # 2. 解析意图
    # 3. 调用工具
    # 4. 生成响应
```

### 讲义 2: 任务规划系统
**关键组件**:
- 待办列表: 简单任务队列
- 任务图: 依赖关系管理
- 子任务分解: 复杂任务拆分

**最佳实践**:
1. 先规划后执行
2. 保持上下文清洁
3. 按需加载技能

### 讲义 3: 内存压缩策略
**三层压缩**:
1. **短期记忆**: 当前会话上下文
2. **中期记忆**: 压缩摘要
3. **长期记忆**: 关键决策点

**实现要点**:
- 定期摘要生成
- 重要性评分
- 选择性保留

### 讲义 4: 并发与协作
**并发模式**:
- 后台线程执行
- 事件通知机制
- 非阻塞操作

**协作架构**:
- 邮箱系统
- 任务板共享
- 协议标准化

## 练习材料

### 练习 1: 基础工具开发
**任务**: 创建一个简单的文件操作 Agent
- 实现文件读取工具
- 实现文件写入工具
- 实现目录列表工具

**要求**:
- 代码行数 < 100
- 支持错误处理
- 提供使用示例

### 练习 2: 任务规划器
**任务**: 构建一个旅行规划 Agent
- 分解旅行任务
- 管理依赖关系
- 处理并行任务

**要求**:
- 使用任务图结构
- 支持任务状态跟踪
- 提供可视化输出

### 练习 3: 内存管理系统
**任务**: 实现三层压缩策略
- 设计摘要算法
- 实现重要性评分
- 测试长期记忆效果

**要求**:
- 压缩率 > 50%
- 关键信息保留 > 90%
- 性能影响 < 10%

### 练习 4: 多 Agent 协作
**任务**: 构建团队协作系统
- 实现邮箱通信
- 设计任务认领机制
- 测试团队协商

**要求**:
- 支持 3+ Agent 协作
- 处理冲突解决
- 提供性能监控

## 测试题目

### 测试 1: 基础知识 (选择题)
1. Agent Loop 的核心组件是什么？
   A) 数据库连接
   B) while 循环 + 工具调度
   C) 神经网络模型
   D) 用户界面

2. 三层压缩策略的目的是什么？
   A) 减少代码行数
   B) 在上下文限制内保持记忆
   C) 提高执行速度
   D) 降低内存使用

### 测试 2: 设计题
**场景**: 你需要设计一个智能客服 Agent 系统

**要求**:
1. 描述系统架构
2. 列出需要的工具
3. 设计任务规划流程
4. 说明内存管理策略

**评分标准**:
- 架构完整性 (30%)
- 工具设计合理性 (25%)
- 流程清晰度 (25%)
- 策略有效性 (20%)

### 测试 3: 编程题
**任务**: 实现一个简单的任务依赖解析器

**输入**: 任务列表和依赖关系
```json
{
  "tasks": ["A", "B", "C", "D"],
  "dependencies": {"B": ["A"], "C": ["A"], "D": ["B", "C"]}
}
```

**输出**: 有效的执行顺序

**要求**:
- 处理循环依赖
- 支持并行任务识别
- 提供执行时间估算

### 测试 4: 案例分析
**案例**: 某公司使用 Agent 系统处理客户支持，但遇到以下问题：
1. 上下文经常丢失
2. 多个 Agent 重复处理同一问题
3. 复杂问题需要人工干预

**问题**:
1. 分析问题原因
2. 提出改进方案
3. 设计实施计划
